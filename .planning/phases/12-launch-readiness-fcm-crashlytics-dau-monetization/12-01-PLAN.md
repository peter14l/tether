# Plan 12-01: Firebase Setup & Telemetry (Crashlytics, DAU)

## Objective
Set up the base Firebase infrastructure and abstract it behind a `TelemetryService` so the UI and domains can log DAU events and non-fatal exceptions without direct SDK coupling.

## Tasks

### 1. Add Firebase Dependencies
<read_first>
- pubspec.yaml
</read_first>
<action>
Add `firebase_core: ^2.24.0`, `firebase_analytics: ^10.7.0`, and `firebase_crashlytics: ^3.4.0` under dependencies in `pubspec.yaml`.
</action>
<acceptance_criteria>
- `pubspec.yaml` contains `firebase_crashlytics`
</acceptance_criteria>

### 2. Implement TelemetryService
<read_first>
- lib/core/telemetry/telemetry_service.dart (New)
</read_first>
<action>
Create `lib/core/telemetry/telemetry_service.dart` with an interface `ITelemetryService` exposing:
`Future<void> logEvent(String name, {Map<String, Object>? parameters});`
`Future<void> logError(dynamic exception, StackTrace stack, {dynamic reason});`
`Future<void> setUserId(String userId);`

Implement `FirebaseTelemetryService` that delegates these calls to `FirebaseAnalytics.instance` and `FirebaseCrashlytics.instance`. Add exception swallowing if `Firebase` is not initialized (so development works smoothly without configs). Register it as a LazySingleton in `lib/injection_container.dart`.
</action>
<acceptance_criteria>
- `telemetry_service.dart` contains `ITelemetryService`
- `injection_container.dart` binds `ITelemetryService` to `FirebaseTelemetryService`
</acceptance_criteria>

### 3. Initialize Catchers in Main
<read_first>
- lib/main.dart
</read_first>
<action>
Update `main.dart` to optionally `await Firebase.initializeApp()` if `DefaultFirebaseOptions.currentPlatform` exists. Since this codebase doesn't have options yet, wrap the initialization in a try-catch block.
Also, add `FlutterError.onError` handler that routes through `getIt<ITelemetryService>().logError` and `PlatformDispatcher.instance.onError`.
</action>
<acceptance_criteria>
- `main.dart` contains `PlatformDispatcher.instance.onError` wiring.
</acceptance_criteria>

## Verification
- Run `flutter analyze` ensuring the telemetry service is cleanly abstracted.
