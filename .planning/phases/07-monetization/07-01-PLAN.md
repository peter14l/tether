<task type="auto">
  <name>Task 2: Create entitlement verification Edge Function</name>
  <files>supabase/functions/verify-entitlement/index.ts</files>
  <action>
Create Edge Function that verifies subscription tier:

1. Accepts: user_id, feature_id (optional - if omitted, returns full entitlement)
2. Checks profile's subscription_tier and subscription_status
3. Applies grace period logic (if status is 'grace_period' and grace_period_until > now(), treat as active)
4. Maps feature_id to required tier:
   - 'unlimited_circles' → requires 'plus'
   - 'couples_features' → requires 'plus'
   - 'family_features' → requires 'plus' or 'family'
   - 'voice_notes' → requires 'plus'
- 'letter_mode' → requires 'plus'
    - 'future_letters' → requires 'plus'
    - 'memory_lane' → requires 'plus'
   - 'ambient_sounds' → requires 'plus'
   - 'badges' → requires 'plus'
5. Returns JSON: { entitled: boolean, tier: string, expires_at: string }

Use Deno and Supabase JavaScript client.
</action>
  <verify>
<automated>supabase functions serve verify-entitlement --no-verify-jwt</automated>
  </verify>
  <done>Function responds with correct entitlement based on subscription tier</done>
</task>

<task type="auto">
  <name>Task 3: Create RevenueCat webhook handler</name>
  <files>supabase/functions/handle-subscription-webhook/index.ts</files>
  <action>
Create Edge Function for RevenueCat webhook:

1. Accepts POST with RevenueCat webhook payload
2. Validates webhook signature using REVENUECAT_WEBHOOK_SECRET
3. Handles event types:
   - INITIAL_PURCHASE: Set tier to 'plus' or 'family', status to 'active', set expires_at
   - RENEWAL: Update expires_at, keep status 'active'
   - CANCELATION: Set status to 'cancelled'
   - EXPIRATION: Set status to 'expired', calculate grace_period_until (now + 7 days)
   - PRODUCT_CHANGE: Handle upgrade/downgrade
4. Updates user profile in Supabase
5. Returns 200 OK to RevenueCat

Log all events to subscriptions_log table for debugging.
</action>

  </verify>
  <done>Webhook processes events and updates user subscription status</done>
</task>
